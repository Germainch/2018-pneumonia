# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Machine learning pipeline for pneumonia detection from chest X-rays using the [RSNA 2018 Pneumonia Detection Challenge](https://www.kaggle.com/c/rsna-pneumonia-detection-challenge) dataset (~30,000 DICOM images).

## Environment Setup

Dependencies are managed via `uv` with `pyproject.toml`. Python ≥ 3.13 required.

```bash
uv sync                   # install dependencies into .venv
uv run jupyter lab        # launch JupyterLab
```

## Key Commands

```bash
# Run dataset summary statistics
uv run python -m lib.dataset_summary

# Re-generate labels.csv from raw annotations (slow, reads all DICOMs)
uv run python -c "from lib.label_images import main; main()"

# Run the full notebook non-interactively
uv run jupyter nbconvert --to notebook --execute 2_knn.ipynb
```

## Architecture

### Data Flow

```
data/mdai_rsna_project_*/   (30k DICOM files)
data/*.json                  (radiologist annotations)
        ↓
lib/label_images.py          → data/labels.csv   (generated once, cached)
        ↓
main.ipynb                   → data/split_cache.npz (image arrays, cached)
        ↓
ML Pipeline: StandardScaler → PCA → ADASYN → KNeighborsClassifier
```

### Labeling Logic (`lib/label_images.py`)

Labels are derived from adjudicated radiologist annotations using majority vote:
- **Sick (1):** "Lung Opacity" annotated at any confidence level
- **Not-sick (0):** "Normal" OR "No Lung Opacity / Not Normal"
- **Ties:** Classified as sick (conservative/clinically safer)
- **QA/Admin labels:** Excluded (84 images skipped)

Output `data/labels.csv` includes DICOM paths, patient metadata (age, sex, view position), binary label, and bounding boxes for opacity regions.

### Image Processing (in `main.ipynb`)

DICOM images are normalized to [0,1], resized to 64×64, and flattened to 4,096-feature vectors. The processed arrays are cached in `data/split_cache.npz` to avoid re-reading DICOMs on subsequent runs.

### Class Imbalance

The dataset is imbalanced (~31% sick, ~69% not-sick). ADASYN oversampling is applied on the training set before fitting the classifier.

## Data Directory

`data/` is gitignored. The directory contains ~4 GB of files:
- `labels.csv` — generated by `lib/label_images.py`
- `split_cache.npz` — cached image arrays from `main.ipynb`
- `mdai_rsna_project_x9N20BZa_images_2018-07-20-153330/` — raw DICOMs
- JSON annotation files from RSNA/MD.ai
